搜索框的样式实现
搜索框用<button>可以很容易地嵌套一张图片在按钮内，但提交不到服务器。
搜索框样式实现的方案：form表单，包裹着输入框和搜索按钮，用form的样式将底色设置为统一颜色并去掉两个input标签的border:0;按钮通过css设为背景图片为图片即可。
input-text鼠标点击输入框会出现蓝色高亮区，用outline:none;去掉样式；“height:24px;line-height:24px;”设置统一可以调整内容垂直方向居中。
input-search按钮上本来是有“提交”文字的，只需要设置value=""即可，图片是通过CSS的background：url(image.jpg) no-repeat;加上的;鼠标滑上去鼠标变为小手是通过cursor:pointer;实现的。
input-text和input-search都是内联块级元素，要想对齐，可以左浮动
解决一个提示问题：浮动边距加倍错误，margin后面跟一句 display:inline;
列表可以直接把列表项图像li{list-style:url(../imag/find.png) none inside;}

搜索框动态效果的实现：JQ技术，点击框框变色，在框中输入内容会弹出搜索提示框，提示框只调用了js还不够，手动改动一些偏移量达到想要的效果;AJAX技术将页面改为动态html，鼠标点击其他位置，提示框就消失了
固定层效果top设置position=fixed后如果不设置偏移量，则是相对于body。设置了top=0则相对于浏览器可视窗口，此时body和mainbody仍然在文档流中，当两个的margin属性相遇时，取外填充较大的那个。所以还要设置一个margin属性将其下拉回来，高度为top的高度（上外填充+上内填充+高度+下内填充+下外填充）
总结头部缺陷：搜索框的图片设置和源网站效果有一定差距；鼠标移动上去，框不会变色，后面列表的图片也不会变色；没有下拉菜单的效果！！！！！

众多li的表现形式体现了类选择器区别去ID选择器的，一个类可以在一个文本内多处使用
overflow:hidden;在父元素中清除浮动（子元素浮动脱离文档流，导致父元素扩展不开）
